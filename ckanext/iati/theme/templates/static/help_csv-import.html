CSV Imports

The registry CSV Import utility allows you to publish, or re-publish, your metadata by uploading a spreadsheet.

The following table details the fields in the CSV file template. For example metadata, we have used the DFID flle for Afghanistan.

FieldLegacy field - not required
NotesExample
registry-publisher-idThis must match your registry ID that your user account has permission to update.dfid
registry-file-idThis must be the id of the package you wish to create or update.dfid-af
titleThe title of this fileDFID Activity File Afghanistan
contact-emailThe specific email contact for this file
source-urlThe URL via which to access this datahttp://iati.dfid.gov.uk/iati_files/Country/DFID-Afghanistan-AF.xml
formatThis will always be IATI-XMLIATI-XML
file-typeState whether this is an activity or organisation file typeactivity
recipient-countryThe country code that applies to this file. NB: not all files are specific to a country. If this is the case, it can be left blankAF
activity-period-startThe start date of the first activity in the file2003-07-01
activity-period-endThe date of the latest activity in the file2016-03-31
last-updated-datetimeThe date the file was last updated. NB: the registry will pick up subsequent update dates via the nightly check, so this field is not essential in the CSV upload.2013-11-29 04:44
